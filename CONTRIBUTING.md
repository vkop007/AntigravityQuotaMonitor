The following is a set of guidelines for contributing to Antigravity Quota Monitor. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

Bugs are tracked as GitHub issues. When creating a bug report, please include as many details as possible:

- **Use a clear and descriptive title** for the issue to identify the problem.
- **Describe the exact steps which reproduce the problem** in as many details as possible.
- **Explain which behavior you expected to see instead and why.**
- **Include screenshots and animated GIFs** which help you demonstrate the steps or the bug.

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues.

- **Use a clear and descriptive title** for the issue to identify the suggestion.
- **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
- **Describe the current behavior and explain which behavior you expected to see instead** and why.
- **Explain why this enhancement would be useful** to most users.

### Pull Requests

- Fill in the required template (if any).
- Do not include issue numbers in the PR title.
- Include screenshots and animated GIFs in your pull request whenever possible.
- Follow the TypeScript style guide.
- Ensure the test suite passes.

## Development Setup

To get started with development:

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/vkop007/antigravity-quota-usage-tracker.git
    cd antigravity-quota-usage-tracker
    ```

2.  **Install dependencies:**

    ```bash
    npm install
    ```

3.  **Open in VS Code:**

    ```bash
    code .
    ```

4.  **Available Scripts:**

    - `npm run compile`: Compiles the TypeScript code.
    - `npm run watch`: Watch for changes and compile automatically.
    - `npm run lint`: Runs ESLint to check for code style issues.
    - `npm run test`: Runs the automated tests.

5.  **Running the extension:**
    - Press `F5` in VS Code to open a new window with the extension loaded.

## Coding Standards

- Use TypeScript for all logic.
- Follow the existing code style and formatting.
- Run the linter before submitting your changes: `npm run lint`.
- Write clear, commented code where necessary.
